# MindRound 极简 MVP 功能清单 (Windows)

## Context

让用户与各种名人独自交流以及拉到一起交流。体验对标微信，极简好用。

**核心价值**: 单聊 + 群聊，本地运行，用户自带 API 密钥

**平台**: Windows 桌面 (Tauri 2.0)

---

## 数据结构

```
%APPDATA%/MindRound/
├── personae/              # 作者库（用户手动复制文件夹到此目录）
│   └── <name>/
│       ├── SKILL.md       # 必需
│       └── avatar.png     # 可选
├── chats/                 # 所有会话
│   └── <uuid>/
│       ├── meta.json      # 会话元数据
│       ├── messages.json  # 消息历史
│       └── memory.md      # 长期记忆
└── settings.json          # API配置
```

## 四个核心页面

### 1. 通讯录 (Contacts)

```
┌─────────────────────────────┐
│ 🔍 搜索作者                  │
├─────────────────────────────┤
│ 📷 乔布斯                    │
│ 📷 马斯克                    │
│ 📷 稻盛和夫                  │
└─────────────────────────────┘
```

| 操作/控件 | 功能 | 能力 |
|-----------|------|------|
| 列表行（作者） | 展示一名已导入作者 | 显示名称（来自目录名或 SKILL.md frontmatter）；头像占位或 avatar.png；**点击进入单聊** |
| 顶部搜索框 | 过滤作者列表 | 按名称模糊搜索；空串显示全部 |
| 按钮「刷新」 | 重新扫描作者库 | 重新读 personae/ 目录；刷新列表 |
| 空状态提示 | 引导用户 | 提示"请将包含 SKILL.md 的文件夹复制到数据目录的 personae/" |

### 2. 会话列表 (Sessions)

```
┌─────────────────────────────┐
│ [+] 新建群聊                 │
├─────────────────────────────┤
│ 📷 乔布斯             你好    │
│    14:23                     │
├─────────────────────────────┤
│ 👥 科技圆桌          三人讨论  │
│    昨天                      │
└─────────────────────────────┘
```

| 操作/控件 | 功能 | 能力 |
|-----------|------|------|
| 列表行（单聊） | 展示与某作者的会话 | 显示作者头像/名称；最后消息预览；最后时间；点击进入聊天 |
| 列表行（群聊） | 展示群会话 | 显示群图标/名称；参与者数量或最后发言者；点击进入聊天 |
| 按钮「[+]」 | 创建新群聊 | 打开多选作者界面；至少选 2 人；输入群名称；创建后进入群聊 |
| 空状态提示 | 引导用户 | 提示"从通讯录选择作者开始聊天" |

### 3. 聊天页面 (Chat - 单聊/群聊共用)

```
┌─────────────────────────────┐
│ ← 乔布斯              ⋯     │
├─────────────────────────────┤
│                    👤 你好   │
│            乔布斯  很高兴见到你│
│                    👤 今天...│
│            乔布斯  让我想想...│
├─────────────────────────────┤
│  输入消息...              发送│
└─────────────────────────────┘
```

| 操作/控件 | 功能 | 能力 |
|-----------|------|------|
| 顶栏返回 | 返回会话列表 | 保留聊天位置（或滚动到底部，二选一） |
| 顶栏标题 | 显示对话对象 | 单聊显示作者名；群聊显示群名称 |
| 消息列表 | 浏览历史 | 加载 messages.json 全部消息；时间戳；用户消息右侧绿色气泡；助手消息左侧白色气泡 |
| 助手消息（单聊） | 阅读作者回复 | 显示作者头像/昵称；流式输出；失败显示错误 |
| 助手消息（群聊） | 区分不同作者 | 每条消息显示对应作者头像/昵称；**不显示轮次** |
| 输入框 | 输入用户消息 | 多行支持；Enter 发送 / Shift+Enter 换行 |
| 按钮「发送」 | 提交消息 | 写入 messages.json；调用 API；流式更新助手消息 |
| 状态「正在输入...」 | 反馈等待 | 请求进行中显示；禁用重复发送 |
| 按钮「⋯」菜单 | 会话操作 | MVP 仅占位；可预留「清空上下文」等 |

```
├─────────────────────────────┤

├─────────────────────────────┤
│ Model                       │

```

| 操作/控件 | 功能 | 能力 |
|-----------|------|------|
| 输入框「Base URL」 | 设置 API 端点 | 保存到 settings.json；格式校验 |
| 输入框「API Key」 | 设置鉴权密钥 | 保存到 settings.json；MVP 明文存储（后续升级） |
| 输入框「Model」 | 设置模型名 | 保存到 settings.json；随请求发送 |
| 按钮「📂」 | 打开数据目录 | 调用系统文件管理器打开 %APPDATA%/MindRound |
| 按钮「保存」 | 持久化配置 | 写入 settings.json；立即生效 |
| 文本「版本号」 | 显示当前版本 | 便于反馈问题 |

## 核心业务规则

### 对话上下文规则

每次调用模型时，发送的上下文由以下组成（按顺序）：

1. **时间窗口**：仅包含**发送请求时刻往前 30 分钟内**的消息（按 timestamp 判断）
2. **memory.md**：当前会话的长期记忆文件
3. **SKILL.md**：当前发言作者的完整人格文件

### 群聊发言顺序

**触发方式**：用户手动发送消息开启一轮对话

**循环模式**（MVP 采用方案 A）：
- 用户创建群聊时选择 2+ 作者，设置群名称
- 用户发送消息后，自动触发一轮循环：按 personae 顺序，A回复 → B回复 → C回复 → ... → 所有作者发言完毕
- 每条助手消息显示对应作者头像/昵称，**不显示轮次**
- 所有作者发言完毕后，等待用户下一条消息开启新一轮循环
- 重启后从 `meta.json` 读取 `currentSpeakerIndex` 恢复状态

**发言状态追踪**：
- `currentSpeakerIndex` 存储在 `meta.json` 中，初始为 0
- 每位作者发言后索引更新，重启后可恢复

**示例流程**（personae = [A, B, C]）：
1. 用户发送消息 → 触发 A 回复 → B 回复 → C 回复
2. C 发言完成后，等待用户下一条消息
3. 用户发送第二条消息 → 再次触发 A → B → C 依次发言
4. 循环往复

### 记忆更新策略

memory.md 的更新时机（MVP 最简方案）：
- 触发：会话空闲超过 5 分钟后，用户发新消息时
- 操作：调用模型摘要历史 → 追加到 memory.md
- 失败处理：不阻塞发送，静默失败

## 文件格式

### settings.json

```json
{
  "apiBaseUrl": "https://api.openai.com/v1",
  "apiKey": "sk-xxx",
  "model": "gpt-4o",
  "dataDir": "C:\\Users\\...\\AppData\\Roaming\\MindRound"
}
```

### meta.json (会话元数据)

```json
// 单聊
{
  "id": "uuid",
  "type": "single",
  "title": "乔布斯",
  "personae": ["steve-jobs"],
  "currentSpeakerIndex": 0,
  "createdAt": "2026-04-15T14:23:56+08:00"
}

// 群聊
{
  "id": "group_uuid",
  "type": "group",
  "title": "科技圆桌",
  "personae": ["elon-musk", "steve-jobs", "turing"],
  "currentSpeakerIndex": 0,
  "createdAt": "2026-04-20T15:00:00+08:00"
}
```

### messages.json (消息历史)

```json
{
  "messages": [
    {"role": "user", "content": "你好", "timestamp": "2026-04-15T14:23:56+08:00"},
    {"role": "assistant", "persona": "steve-jobs", "content": "很高兴见到你", "timestamp": "2026-04-15T14:24:10+08:00"}
  ]
}
```

### memory.md (长期记忆)

```markdown
# 对话记忆

- 用户对设计哲学感兴趣
- 讨论过专注力的重要性
...
```

### SKILL.md (作者人格)

```markdown
---
name: 乔布斯
description: 苹果联合创始人
---

你是一位产品设计师，崇尚简约和专注...
```

## MVP 验收清单

- [ ] 应用启动，创建默认数据目录
- [ ] 设置页可配置 API
- [ ] 手动复制 personae 文件夹后，刷新通讯录可见
- [ ] 从通讯录点击作者，进入单聊
- [ ] 发送消息，收到流式回复
- [ ] 关闭重启，历史消息仍在
- [ ] 创建群聊，选 2+ 作者
- [ ] 群聊中多名作者依次回复，每条显示头像/昵称
- [ ] 上下文符合 30 分钟窗口规则
- [ ] 可打开数据目录
