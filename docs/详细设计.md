# MindRound 详细设计

## 设计原则

1. **接口隔离**: 为未来功能预留接口，MVP 实现最小版本
2. **分层架构**: 核心逻辑与平台无关，UI 层可替换
3. **依赖注入**: 便于后续扩展（如导入、存储后端）
4. **事件驱动**: 支持未来异步操作（如网络导入、自动更新）

## 数据流设计

### 单聊数据流

```
┌─────────┐    ┌─────────┐    ┌─────────────┐    ┌─────────┐
│  User   │───▶│ ChatUI  │───▶│ ChatService │───▶│  API    │
└─────────┘    └─────────┘    └─────────────┘    └─────────┘
                                   │                   │
                                   ▼                   ▼
                            ┌─────────────┐    ┌─────────────┐
                            │ ContextBuilder│  │   LLM       │
                            └─────────────┘    └─────────────┘
                                   │                   │
                                   │◀──────────────────┘
                                   ▼
                            ┌─────────────┐
                            │ChatRepo     │
                            │+write msg   │
                            └─────────────┘
```

### 发送消息详细流程

```
1. 用户输入消息 → ChatUI
2. ChatUI → ChatService.sendMessage()
3. ChatService:
   a. 调用 ContextBuilder.buildContext()
      - 从 ChatRepo 获取近30分钟消息
      - 从 ChatRepo 获取 memory.md
      - 从 PersonaRepo 获取 SKILL.md
   b. 构造 API 请求
   c. 调用 ApiRepository.chat()（流式）
4. 流式响应:
   - 每个chunk → 更新 UI
   - 完成后 → 写入 ChatRepo
5. (异步) 触发 MemoryService.summarizeAndSave()
```

### 群聊数据流

```
用户发送消息
    │
    ▼
ChatService.sendMessage()
    │
    ├──▶ 保存用户消息到 messages.json
    │
    ├──▶ 获取 currentSpeakerIndex（从 meta.json）
    │
    ├──▶ 计算下一位发言者：nextIndex = (currentSpeakerIndex + 1) % personaIds.length
    │
    ├──▶ ContextBuilder.buildForGroup()
    │        - 群共享 memory.md
    │        - 当前发言作者的 SKILL.md
    │        - 近30分钟群消息
    │
    ├──▶ ApiRepository.chat() → 当前作者回复
    │
    ├──▶ 保存助手消息到 messages.json
    │
    ├──▶ 更新 currentSpeakerIndex 到 meta.json
    │
    └───▶ 如果所有作者都已发言一轮 → 结束，等待用户下一条消息
           否则 → 继续循环触发下一个作者
```

## 伪代码

### ChatService.sendMessage()

```typescript
class ChatService {
  async sendMessage(chatId: string, content: string): Promise<void> {
    // 1. 创建用户消息
    const userMessage: Message = {
      role: 'user',
      content,
      timestamp: new Date(),
    };

    // 2. 立即写入存储（乐观更新）
    await this.chatRepo.addMessage(chatId, userMessage);

    // 3. 获取会话信息
    const chat = await this.chatRepo.findById(chatId);
    if (!chat) throw new Error('Chat not found');

    // 4. 构建上下文
    const context = await this.contextBuilder.buildForChat(chat);

    // 5. 调用 API（流式）
    const assistantMessage: Message = {
      role: 'assistant',
      personaId: chat.personaIds[0],  // 单聊只有一个作者
      content: '',
      timestamp: new Date(),
    };

    // 6. 流式处理
    for await (const chunk of this.apiRepo.chat({ messages: context.messages })) {
      assistantMessage.content += chunk;
      // 通知 UI 更新（通过 Store 或 Event）
      this.emit('messageUpdating', { chatId, message: assistantMessage });
    }

    // 7. 完成后写入
    await this.chatRepo.addMessage(chatId, assistantMessage);

    // 8. 异步更新记忆（不阻塞）
    this.memoryService.summarizeAndSave(chat).catch(err => {
      console.error('Memory update failed:', err);
    });

    // 9. 如果是群聊，触发下一个作者
    if (chat.type === 'group') {
      await this.continueGroupChat(chat);
    }
  }

  private async continueGroupChat(chat: Chat): Promise<void> {
    // 获取当前发言者索引（从 meta.json 持久化存储）
    let currentIndex = await this.chatRepo.getSpeakerIndex(chat.id);
    const personaCount = chat.personaIds.length;

    // 循环触发所有作者依次发言
    for (let i = 0; i < personaCount; i++) {
      const nextIndex = (currentIndex + 1) % personaCount;
      const nextPersonaId = chat.personaIds[nextIndex];

      // 构建上下文并触发当前作者回复
      await this.triggerAssistantReply(chat.id, nextPersonaId);

      // 更新发言索引并持久化
      currentIndex = nextIndex;
      await this.chatRepo.updateSpeakerIndex(chat.id, currentIndex);
    }
    // 一轮结束后，等待用户下一条消息开启新一轮循环
  }

  private async triggerAssistantReply(chatId: string, personaId: string): Promise<void> {
    // 1. 构建上下文（该作者的 SKILL + 30分钟窗口 + memory）
    const chat = await this.chatRepo.findById(chatId);
    if (!chat) return;

    const context = await this.contextBuilder.buildForGroup(chat, personaId);

    // 2. 调用 API（流式）
    const assistantMessage: Message = {
      role: 'assistant',
      personaId,
      content: '',
      timestamp: new Date(),
    };

    for await (const chunk of this.apiRepo.chat({ messages: context.messages })) {
      assistantMessage.content += chunk;
      this.emit('messageUpdating', { chatId, message: assistantMessage });
    }

    // 3. 保存助手消息
    await this.chatRepo.addMessage(chatId, assistantMessage);
  }
}
```

### ContextBuilder.buildForChat()

```typescript
class ContextBuilderService {
  async buildForChat(chat: Chat): Promise<Context> {
    const now = new Date();
    const thirtyMinutesAgo = new Date(now.getTime() - 30 * 60 * 1000);

    // 1. 获取时间窗口内的消息
    const allMessages = await this.chatRepo.getMessages(chat.id);
    const recentMessages = allMessages.filter(
      msg => msg.timestamp >= thirtyMinutesAgo
    );

    // 2. 获取长期记忆
    const memory = await this.chatRepo.getMemory(chat.id);

    // 3. 获取作者 SKILL
    const personaId = chat.personaIds[0];
    const skill = await this.personaRepo.getSkillContent(personaId);

    return {
      messages: recentMessages,
      memory,
      skill,
    };
  }
}
```

### MemoryService.summarizeAndSave()

```typescript
class MemoryService {
  async summarizeAndSave(chat: Chat): Promise<void> {
    // 1. 检查是否需要更新（策略可配置）
    if (!await this.shouldUpdateMemory(chat)) {
      return;
    }

    // 2. 获取历史消息
    const messages = await this.chatRepo.getMessages(chat.id);
    const currentMemory = await this.chatRepo.getMemory(chat.id);

    // 3. 调用 LLM 摘要
    const summary = await this.apiRepo.chatComplete({
      messages: [
        {
          role: 'system',
          content: `你是一名记忆整理助手。根据对话历史，提取关键信息追加到现有记忆中。
          现有记忆：\n${currentMemory}\n\n
          新对话：\n${this.formatMessages(messages)}\n\n
          请输出更新后的完整记忆（Markdown 格式）。`,
        },
      ],
    });

    // 4. 保存
    await this.chatRepo.updateMemory(chat.id, summary);
  }

    const lastMessageTime = new Date(lastMessage.timestamp);
    const now = new Date();
```

## 关键决策记录

| 决策 | 理由 |
|------|------|
| Repository 模式 | 隔离存储实现，便于未来切换到数据库/云存储 |
| Platform Adapter 接口 | 多平台支持，不污染核心业务逻辑 |
| 时间窗口 + memory.md | 平衡上下文完整性和 token 效率 |
| 流式响应 | 更好的用户体验，避免长时间等待 |
| 异步记忆更新 | 不阻塞主流程，失败不影响对话 |
| Zustand | 轻量级，适合中小型应用状态管理 |
